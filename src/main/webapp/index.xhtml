Library

Website features

Goal of the website:

The purpose of the website is to design and implement an imaginary library rental registry software using Java ServerFaces, Primefaces, and MySQL technologies. Books and other multimedia materials (CDs, software) can be borrowed by registered members in the library. The site is used by the library manager and staff to register enrollments and check member rentals, and to query members who did not return borrowed materials on time.

Main features

Member list

  • The system keeps track of all the members of the library.
  • Option to add new members.
  • Search members based on name, ID, or address.
  • Modify members' data or delete an existing member.

Stock inventory

  • Keep an inventory of all items in the library.
  • Keep the status (available, borrowed) of stocks up-to-date.
  • Search items in the inventory.

Borrows

Managing borrows. Members can only have 6 active items borrowed at a time, and only active members can borrow.

Returned stock

The user of this site can record when a stock is returned. The date of return is automatically inserted, and the number of copies also increases when a copy is returned.

Query past due borrows

The user can query items that is borrowed to a member for the entered number of days, and still haven't been brought back.
